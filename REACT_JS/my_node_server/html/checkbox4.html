Un cluster

  1. un équipement, relié au réseau dont la principale fonction est le stockage de données
  2. un réseau de stockage
  3. peut-être un hub
  4. peut-être un switch
  5. une grappe de serveurs constitué de 2 serveurs au minimum
  6. peut-être un routeur

    L'interopérabilité entre les composants d'un SI distribué est :

  1. la capacité de ces composants à être remplacés sans interruption de l’informatique.
  2. la capacité de ces composants à échanger des services et des données.
  3. la capacité de ces composants à traiter des données.

    Les 3 informations d'interopérabilité :

  1. Protocole de communication entre Client/Serveur, système d’exploitation, Base de données
  2. Protocole de communication entre Client/Serveur, Intranet, Extranet
  3. Protocole de communication entre Client/Serveur, Interface de définition d’échanges, Annuaire référençant le service échangé

    Les Architectures Distribuées

  1. Un système d'information ou un réseau pour lequel l'ensemble des ressources disponibles ne se trouvent pas au même endroit ou sur la même machine.
  2. Un système d'information ou un réseau pour lequel l'ensemble des ressources disponibles se trouvent au même endroit ou sur la même machine.
  3. Un système d'information ou un réseau pour lequel l'ensemble de la topologie du réseau utilisée ne se trouvent pas au même endroit ou sur la même machine.
  4. Matérielle, Liaison, Réseau, Transport, Session, Présentation, Application
  5. Physique, Liaison, Réseau, Transport, Session, Présentation, Haute

    Modèle du Gartner Group est un modèle à :

  1. Architecture 3-tiers
  2. Architecture 2-tiers
  3. Architecture n-tiers

    Tout programme applicatif contient trois types de traitements :

  1. Liés aux métiers, liés à la présentation, liés aux protocoles de communication
  2. Liés aux serveurs, liés à la présentation, liés aux données manipulées
  3. Liés aux métiers, liés à la présentation, liés aux données manipulées

    Les trois composants essentiels d’une architecture Web sont :

  1. Navigateur Web, Serveur Web, Page Web
  2. Système d’exploitation, Service Web, Page Web
  3. Navigateur Web, Système Web, Application Web

    La sécurité

  1. Firewall, proxy-HTTP, cryptage
  2. Firewall, 4-tiers, cryptage
  3. Firewall, proxy-HTTP, SMTP

    Architecture N-Tiers

  1. Utilisateur, Ordinateur, Serveur
  2. Client, Middleware, DB
  3. Client, Serveur, Page Web

    Les modèles informatiques du cloud sont :

  1. IaaS, PaaS, SaaS
  2. LaaS, PaaS, SaaS
  3. IaaS, TaaS, SaaS
